Good Luck Is Coming

Introduction

Developed in one month for the 2026 NetEase Game Jam by an 8-person team, Good Luck Is Coming is a 2D truck management game where players dispatch trucks to various locations, switching to a top-down perspective to aim and crash onto shifting, yellow-highlighted targets.

I am responsible for:

  • Programming the game, notably UI interactions and resource creation
  • Technical Design on game's core systems and mechanics, understanding and deciding if some mechanisms are feasible
  • Thinking about customs events that make the game interesting
  • UI Design and UX exploration, understanding how users interact with the game

Scripting

The game is entirely coded in Godot using the gdscript language. I haven't created a management simulation game before, which usually has many UI elements. I also never worked with another coder simultaneously in a game jam before.

For the UI elements, I made sure to make use of Horizontal and Vertical Containers to make sure the UI elements are responsive to different screen sizes and are organized properly. I also made sure signals are used appropriately to communicate between UI elements and provide interation. Due to the game having a variety of targets, trucks and civilians, I made sure to use resources for each category of objects to make sure the game is scalable.

Game Design

While the core mechanics of the game is designed by other game designers, I contributed on designing custom events for the game to make it more interesting.

Initally, traffic congestion event was designed to slow down trucks and even block paths. making reaching the target destination more interesting. However, not only it will it be difficult to implement, it also will only affect the truck routing gameplay and not the truck collision gameplay.

Therefore I designed earthquake and flooding events instead. Not only will it slow down the trucks during truck routing, weird effects are created to spice up the truck collision gameplay. Details can be seen in the videos below.

UX Design

As the main UI Programmer of the game, I am also responsible for designing the core UX and UI elements of the game. This includes drawing diagrams with iterations, serving as prototypes. This is important for the team to understand how the player will interact with the game, and how the game will respond to the player's actions.

Diagrams are drawn in Miro before UI implementation as shown below. Changes are made to the main UI due to the specification of the game changed to having only 2 targets appearing at the same time instead of multiple targets appearing at different sequences.

Learning Outcomes

I learnt how to work with another programmer as effectively as possible, by not only using version control, but also by communicating with each other on what to work on. I also learnt how to maintain flexibility in the game design process, thinking about game design as a more iterative process rather than a waterfall model.

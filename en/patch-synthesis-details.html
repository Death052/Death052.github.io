Patch Based Synthesis

Introduction

This project is my first Unity project dedicated to Compute Shaders, and it serves as my Computer Science Undergraduate Final Year Project. The main goal of this project is to implement a patch-based synthesis algorithm, which is a texture synthesis algorithm that takes in a sample texture, and generates a new texture of any size that looks similar to the sample texture.

Programs Used

  • Unity

Algorithm Logic

The algorithm first takes in a sample texture, and it divides the sample texture into small square patches. The boundary pixels of each patch is then saved, which is used to calculate the similarity between patches.

The algorithm then starts creating the new texture by randomly picking a patch from the sample texture, and then adding new patches based on the boundary pixels of the existing patches. This algorithm is quite complicated, therefore you are welcome to view the full document I wrote if you are interested.

Learning Outcomes

I have learnt a lot about compute shaders through this project, and understood how using compute shaders can offset heavy computation from the CPU to the GPU, significantly improving the performance of certain aspects of the algorithm.

Coincidentally, this my first ever Unity project (or game project in general) that utilize 3D elements. Therefore, I learnt how to use 3D raycasting in Unity as well as using some 3D materials.

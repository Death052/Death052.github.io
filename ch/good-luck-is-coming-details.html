《好运来》 (Good Luck Is Coming)

游戏简介

《好运来》是由 8 人团队历时一个月为 2026 年网易 Game Jam 开发的 2D 卡车管理游戏。玩家需要派遣卡车前往各个地点，并切换至俯视视角进行瞄准，冲撞不断变换且带有黄色高亮标示的目标。

我在此项目中负责：

  • 游戏程序编写，特别是 UI 互动与资源（Resource）建立
  • 核心系统与机制的技术设计，评估并决定部分机制的实作可行性
  • 构思让游戏更具趣味性的自订随机事件
  • UI 设计与 UX 探索，深入理解玩家如何与游戏进行互动

程序编写

游戏完全使用 Godot 的 GDScript 语言进行开发。在此之前，我从未制作过通常包含大量 UI 元素的经营模拟类游戏，同时也是第一次在 Game Jam 中与另一位设计师协同协作。

在 UI 元素方面，我善用了水平和垂直容器（Horizontal & Vertical Containers），以确保 UI 界面能适应不同的屏幕尺寸并保持井然有序。此外，我也确保正确运用了信号（Signals）系统来建立 UI 元素之间的沟通与互动。由于游戏中存在多种不同的目标、卡车和路人，我为每一类物件建立了独立的资源档案（Resources），以确保游戏具备良好的可扩充性。

机制设计

虽然游戏的核心机制是由其他游戏设计师规划，但我为游戏构思并设计了自订随机事件，以增添游戏的趣味性。

起初，团队设计了「交通堵塞」事件来减缓卡车速度甚至阻断路径，试图让前往目标的过程更有挑战性。然而，这项机制不仅实作难度高，且只会影响卡车路线规划的玩法，对核心的卡车冲撞玩法毫无帮助。

因此，我改为设计了「地震」和「洪水」事件。这两项事件不仅能在路线规划阶段延缓卡车，还能在冲撞阶段创造出独特的异常效果，为玩法注入变化。详细效果可参考下方影片。

UI/UX 设计

作为游戏的主要 UI 程序员，我也负责规划核心 UX 和 UI 元素。这包括绘制多个版本的迭代图表作为原型（Prototype）。这对于团队理解玩家将如何与游戏互动，以及游戏如何回应玩家的操作至关重要。

学习成果

我学会了如何更高效地与另一位程序员协同工作，不仅通过版本控制系统，还包括主动与彼此沟通各自的工作分配。同时，我也学会了在游戏设计过程中保持弹性，将游戏设计视为一种反复修改的迭代过程，而非传统僵化的瀑布流模型。

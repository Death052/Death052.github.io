Blossems of the Dead

简介

《Blossems of The Dead》是我在伦敦金史密斯学院（Goldsmiths University）游戏设计课程中正在进行的毕业设计作品。 这款游戏是一款基于 Unreal Engine 开发的 3D 解谜动作平台游戏。我正与另外 3 名团队成员一同制作这款游戏。

主角穿梭于亡者之地，在不同的灵魂形态之间切换以解开谜题，最终找到她已故的祖母，向她作最后的告别。

我负责的工作包括：

  • 编写游戏的核心机制与系统代码
  • 设计涉及灵魂切换的游戏机制
  • 游戏界面的 UI/UX 设计
  • 通过角色对话及整体故事进行叙事设计

程序脚本 (Scripting)

我的职责

作为本项目的程序开发者，我主要使用 Blueprint（蓝图）实现游戏中的大部分机制。 同时我也担任技术设计师的角色，确保游戏机制可以重复使用，并能以合理的成本与精力实现。

作为一款具备状态切换功能的解谜平台游戏，我实现了一个状态切换系统，只需按下按钮即可将玩家传送于两个几乎完全相同的世界之间。 每个世界都拥有稍有不同的物件和路径供玩家探索。某些物件在不同状态下互动时甚至会呈现不同的特性。

我还实验了 Niagara 粒子系统与不同的材质来制作风效，并编写了用于物件轮廓外框效果的 Shader 着色器。

版本控制

本项目使用了版本控制，主要采用 Perforce 作为版本控制软件。我负责协助搭建 Perforce 服务器，并协助团队成员设置其客户端。

机制设计 (Mechanic Design)

我的职责

我的任务是构思有趣的机制设计，以增加游戏的丰富度与多样性，避免它变成单纯的“步行模拟器”。 尽管并非所有想法最终都会被采纳和实现，但每一个点子都是经过流程图绘制与详细思考的。

由于状态切换被确定为游戏的主要机制，因此我提出了许多围绕该概念延伸的子机制。 鉴于状态切换机制基于“Sol Flames（灵魂之火）”，我思考了火焰如何会被风吹熄、以及如何点亮物件，并基于这些概念进行扩展。 下方的图表仅展示了我构思的部分想法。

UI/UX 设计

我的职责

我负责规划与制作游戏的 UI 界面。目前已完成游戏内 overlay UI 的版面配置。 未来我还将规划主菜单、暂停菜单以及对话系统的界面配置。

我的想法

与我平时规划 UI 设计的方式类似，我会先从带有注解的 Wireframe（线框图）开始，在获得团队一致同意后，才将设计落实至最终界面。 最初团队曾讨论过让玩家在 Sol Flame 状态下拥有冷却时间与持续时间，因此我曾规划游戏 UI 包含冷却时间条。 不过这个设定后来被废弃了，我转而采用了更简洁的 UI 覆盖层设计。我也开始制作了暂停/设置菜单的原型。

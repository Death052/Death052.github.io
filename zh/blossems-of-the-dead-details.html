Blossems of the Dead

簡介

《Blossems of The Dead》是我在倫敦金匠學院（Goldsmiths University）遊戲設計課程中正在進行的畢業設計作品。 這款遊戲是一基於 Unreal Engine 開發的 3D 謎題動作平台遊戲。我正與另外 3 名團隊成員一同製作這款遊戲。

主角穿梭於亡者之地，在不同的靈魂形態之間切換以解開謎題，最終找到她已故的祖母，向她作最後的告別。

我負責的工作包括：

  • 編寫遊戲的核心機制與系統程式碼
  • 設計涉及靈魂切換的遊戲機制
  • 遊戲介面的 UI/UX 設計
  • 透過角色對話及整體故事進行敘事設計

程式腳本 (Scripting)

我的職責

作為本項目的主要程式設計師，我主要使用 Blueprint（藍圖）實現遊戲中的大部分機制。 同時我也擔任技術設計師的角色，確保遊戲機制可以重複使用，並能以合理的成本與精力實現。

作為一款具備狀態切換功能的解謎平台遊戲，我實作了一個狀態切換系統，只需按下按鈕即可將玩家傳送於兩個幾乎完全相同的世界之間。 每個世界都擁有稍有不同的物件和路徑供玩家探索。某些物件在不同狀態下互動時甚至會呈現不同的特性。

我還實驗了 Niagara 粒子系統與不同的材質來製作風效，並編寫了用於物件輪廓外框效果的 Shader 著色器。

版本控制

本項目使用了版本控制，主要採用 Perforce 作為版本控制軟體。我負責協助架設 Perforce 伺服器，並協助團隊成員設定其客戶端。

機制設計 (Mechanic Design)

我的職責

我的任務是構思有趣的機制設計，以增加遊戲的豐富度與多樣性，避免它變成單純的「步行模擬器」。 儘管並非所有想法最終都會被採納和實作，但每一個點子都是經過流程圖繪製與詳細思考的。

我的想法

由於狀態切換被確定為遊戲的主要機制，因此我提出了許多圍繞該概念延伸的子機制。 鑑於狀態切換機制基於「Sol Flames（靈魂之火）」，我思考了火焰如何會被風吹熄、以及如何點亮物件，並基於這些概念進行擴展。 下方的圖表僅展示了我構思的部分想法。

UI/UX 設計

我的職責

我負責規劃與製作遊戲的 UI 介面。目前已完成遊戲內 overlay UI 的版面配置。 未來我還將規劃主選單、暫停選單以及對話系統的介面配置。

我的想法

與我平時規劃 UI 設計的方式類似，我會先從帶有註解的 Wireframe（線框圖）開始，在獲得團隊一致同意後，才將設計落實至最終介面。 最初團隊曾討論過讓玩家在 Sol Flame 狀態下擁有冷卻時間與持續時間，因此我曾規劃遊戲 UI 包含冷卻時間條。 不過這個設定後來被廢棄了，我轉而採用了更簡潔的 UI 覆蓋層設計。我也開始製作了暫停/設定選單的原型。

Stitched.EXE

簡介

《Stitched.EXE》是一款沉浸式恐怖平台遊戲，主角為兩位任天堂角色——敲冰塊（Ice Climbers）。這款遊戲的遊玩時間約為 20 分鐘，是我個人首次成功獨立製作並公開發行的單人遊戲專案。遊戲講述了兩位敲冰塊主角試圖在一個擁有他們已故手足外貌的超自然存在手中倖存下來的故事。

我主要負責：

  • 編寫整個遊戲的程式，實現所有規劃的功能
  • 關卡設計規劃
  • 設計並拓展躲藏機制
  • 遊戲美術創作

使用的軟體

Unity 用於製作這款遊戲的程式開發軟體
Clip Studio Paint 用於創作遊戲的美術資產
Figma 使用者介面（UI）規劃
HitFilm 用於製作宣傳素材

程式編寫

整個遊戲是在 Unity 引擎中以 C# 語言編寫而成，由許多銜接在一起的場景所組成。遊戲具備了完整的 UI 選單系統以及流暢的平台動作系統。

由於這是一個個人專案，遊戲內的所有程式編寫均由我獨立完成。我分析並採用了一套利用重力和向量操作來提升打擊感（feel-good）的平台跳躍教學，並設計了一個簡單的攻擊敵人系統。這也是我第一次利用對話陣列（arrays of dialogues）來實現對話系統。此外，我還為敵人引入了有限狀態機（state machines），讓它們能擁有複雜且自定義的行為模式。

機制設計

這款遊戲的玩法並不依賴簡單的戰鬥。由於兩位主角不斷受到一個古神恐怖存在（eldrith horror）的追殺，我希望在遊戲中融入捉迷藏的元素，以營造出角色的懸疑感與無助感。遊戲中有兩個部分設計了帶有微調變化的躲藏機制。

第一部分引入了一個融入背景的巨大懸浮頭顱。這個懸浮頭顱會在設定的時間內睜開眼睛並環顧地圖。第二部分則引入了一隻會在部分地圖上行走並能四處傳送的怪物，玩家只能透過節奏性的心跳聲來判斷怪物的距離。在這兩種情況下，玩家都需要躲在物體後面以避免瞬間死亡。

UI/UX 設計

在實際實裝之前，UI 已先使用 Figma 進行了預先規劃。 在第一次迭代中，暫停選單被設計得很簡單，僅保留最少元素，但測試玩家反映外觀顯得過於單調。 因此，我參考了 UI 資料庫中的範例，將敲冰塊角色的視覺元素融入設計中並重新調整了佈局。我還根據使用者的反饋，新增了全螢幕切換按鈕。

美術設計

我使用 Clip Studio Paint 來繪製遊戲內的所有資產。 我特別參考了遊戲《小小夢魘》（Little Nightmares）的配色方案，該遊戲與本專案有著相似的敘事氛圍。我在遊戲的第一部分使用了較冷淡的色調，在第二部分則使用了詭異的暖色調，這兩種場景都在傳遞一種絕望感。

學習成果

作為我人生中建立的第一個獨立單人遊戲專案，我學會了如何管理遊戲創意，並將其從一個簡單的平台跳躍概念擴展為具有多種躲藏機制的複雜遊戲。同時，我也深入學習了 Unity 引擎與 C# 程式編寫。

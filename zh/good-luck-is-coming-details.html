《好運來》 (Good Luck Is Coming)

遊戲簡介

《好運來》是由 8 人團隊歷時一個月為 2026 年網易 Game Jam 開發的 2D 卡車管理遊戲。玩家需要派遣卡車前往各個地點，並切換至俯視視角進行瞄準，衝撞不斷變換且帶有黃色高亮標示的目標。

我在此專案中負責：

  • 遊戲程式編寫，特別是 UI 互動與資源（Resource）建立
  • 核心系統與機制的技術設計，評估並決定部分機制的實作可行性
  • 構思讓遊戲更具趣味性的自訂隨機事件
  • UI 設計與 UX 探索，深入理解玩家如何與遊戲進行互動

程式編寫

遊戲完全使用 Godot 的 GDScript 語言進行開發。在此之前，我從未製作過通常包含大量 UI 元素的經營模擬類遊戲，同時也是第一次在 Game Jam 中與另一位程式設計師協同協作。

在 UI 元素方面，我善用了水平和垂直容器（Horizontal & Vertical Containers），以確保 UI 介面能適應不同的螢幕尺寸並保持井然有序。此外，我也確保正確運用了訊號（Signals）系統來建立 UI 元素之間的溝通與互動。由於遊戲中存在多種不同的目標、卡車和路人，我為每一類物件建立了獨立的資源檔案（Resources），以確保遊戲具備良好的可擴充性。

機制設計

雖然遊戲的核心機制是由其他遊戲設計師規劃，但我為遊戲構思並設計了自訂隨機事件，以增添遊戲的趣味性。

起初，團隊設計了「交通堵塞」事件來減緩卡車速度甚至阻斷路徑，試圖讓前往目標的過程更有挑戰性。然而，這項機制不僅實作難度高，且只會影響卡車路線規劃的玩法，對核心的卡車衝撞玩法毫無幫助。

因此，我改為設計了「地震」和「洪水」事件。這兩項事件不僅能在路線規劃階段延緩卡車，還能在衝撞階段創造出獨特的異常效果，為玩法注入變化。詳細效果可參考下方影片。

UI/UX 設計

作為遊戲的主要 UI 程式設計師，我也負責規劃核心 UX 和 UI 元素。這包括繪製多個版本的迭代圖表作為原型（Prototype）。這對於團隊理解玩家將如何與遊戲互動，以及遊戲如何回應玩家的操作至關重要。

學習成果

我學會了如何更高效地與另一位程式設計師協同工作，不僅透過版本控制系統，還包括主動與彼此溝通各自的工作分配。同時，我也學會了在遊戲設計過程中保持彈性，將遊戲設計視為一種反覆修改的迭代過程，而非傳統僵化的瀑布流模型。
